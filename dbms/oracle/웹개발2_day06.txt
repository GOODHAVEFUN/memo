JDK : 자바 개발 도구
	자바 어플리케이션을 개발하기 위한 도구를 제공함
	자바코드를 작성하고 컴파일하여 실행가능한 형태로 만들기 위한 도구

	자바 컴파일러(javac) : 소스코드를 바이트코드로 변환하는역할
	실행파일(.java) : 바이트 코드를 JVM에서 실행하기 위한 도구
	디버깅(jdb) : 자바 프로그램의 디버깅을 지원
	개발도구(javadoc) : 소스코드 기반으로 API문서를 생성

https://docs.oracle.com/en/java/javase/11/tools/tools-and-command-reference.html#GUID-55DE52DF-5774-4AAB-B334-E026FBAE6F34
		
JRE : 자바 실행 환경,자바어플리케이션을 실행하기 위한 구성요소 포함
	JVM : 소스코드를 컴파일한 바이트 코드 실행함
	런타임 라이브러리 : 자바 어플리케이션 실행에 필요한 클래스 라이브러리
	-개발자가 자바 어플리케이션을 개발하지않고 실행만 하는 경우 JRE만 설치하면 됨

JVM : 자바 가상 머신,자바어플리케이션을 실행하기위한 가상 컴퓨터 
	자바 바이트 코드를 기계어로 변환하고 실행하는 역할
	플랫폼 독립성을 제공(이식성이 좋다) 
	운영체제와 하드웨어 상관없이  동일한 방식으로 자바어플리케이션을 실행가능
=> JDK 자바개발도구 를 제공하고 JRE, JVM 을 포함
	JRE는 자바 어플리케이션 실행환경을 제공하며, 
	JVM은 자바 바이트코드를 실행하는 가상 머신 입니다

JAVA 코드의 컴파일 과정
	소스코드 작성 -> 자바 컴파일러(javac)를 사용하여 java소스를 컴파일한다
	//컴파일러는 소스코드의 문법 검사, 바이트코드로 변환함 .class 가진 클래스파일로 저장됨
	-> 바이트코드 생성(JVM) 
	-> 클래스로더를 사용하여 필요한 클래스 파일을 로드(JVM의 메모리 영역중 method영역에 저장) 
	-> 메모리 할당(stack , heap 등)
	-> 로드된 클래스 파일릐 바이트 코드를 검증
	-> 실행엔진(바이트코드 실행)

	=> 소스코드 작성 후 컴파일러를 사용 후 바이트코드로 변환하고 클래스파일로 생성된 뒤
	JVM이 해당 클래스 파일을 로드하여 실행하는 과정입니다

제약조건
	1. PRIMARY KEY(PK)
		고유한 값이며 각 행의 구분점으로 사용된다
		중복이 없고 NULL값을 허용하지 않는다
	2. FOREIGN KEY(FK)
		다른 테이블의 PK를 사용하며 중복이 가능하다
		보통 테이블끼리 관계를 맺을 때 사용한다
		NULL 허용하지만 좋은 상황이 아니다
	3. UNIQUE KEY(UK)
		NULL은 허용하지만 중복을 허용하지 않는다
	
DDL(Data Definition Language) : 데이터 정의어
	테이블 조작, 제어 관련 쿼리문
CREATE : 테이블 생성
	CREATE TABLE 테이블명(컬럼명 자료형(용량) [제약조건],...1);

DROP : 테이블 삭제
	DROP TABLE 테이블명;

ALTER : 테이블 수정
	ALTER TABLE 테이블명
		- 테이블명 수정 : RENAME TO 새로운테이블명;
		- 칼럼 추가 : ADD(새로운컬럼명 자료형(용량));
		- 컬럼명 변경 : RENAME COLUMN 기존 컬럼명;
		- 컬럼명 삭제 : DROP COLUMN 기존 컬럼명
		- 컬럼 타입 수정 : MODIFY(기존컬럼명 자료형(용량));

TRUNCATE : 테이블 내용 전체 삭제
   TRUNCATE TABLE 테이블명;
